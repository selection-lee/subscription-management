## 0002 · 알림 — MVP는 인앱 only ("놓친 알림 재조립")

- **날짜**: 2026-06-28
- **상태**: 합의·구현됨 (커밋 3e91dad)
- **관련**: 요구사항 §7.1(알림=전역 on/off·시간·1일전/당일), §8-16(재조립 방식), §7.3(알림 센터=Backlog)

### 결정
실제 OS/웹 푸시 없이 **인앱으로만** 알림 제공.
- `paymentAlerts(subs, daysBefore)`: 결제일이 **지났거나(놓침)** `daysBefore` 이내로 임박한 활성 구독을 계산(순수 함수). 가장 급한 순 정렬.
- 화면: 홈 "결제 알림 N건" + 읽기전용 `/alerts`(지남/오늘/내일 라벨). 설정에 전역 on/off + 기본 시간.
- "놓친(지난) 결제 포함"은 의도적 — 현재 데이터에 지난 결제가 많고, §8-16 "앱 열었을 때 재조립" 취지와 맞음.

### 제외 (이유)
- **실제 푸시(Service Worker + Web Push)**: Phase 3. 권한·SW·푸시 스케줄러(백엔드 cron) 필요. 오프라인 우선 원칙과도 충돌.
- **알림 센터(미확인/확인됨 탭·검토완료·스누즈)**: Backlog(§7.3).
- **커스텀 오프셋·프리셋·자동적용**: Phase 2(§7.2).

### 향후
실제 푸시 도입 시: NotificationSetting의 `daysBefore`/`onPaymentDay`를 UI로 노출, 푸시 구독 저장 + 발사 스케줄러 추가. `paymentAlerts`는 "무엇을 알릴지" 계산으로 재사용.

## 0001 · 통계의 통화 처리 — "B 엔진 + A 뷰"

- **날짜**: 2026-06-28
- **상태**: 합의됨 (MVP `/stats` 착수 직전)
- **관련**: 요구사항 §7.1(통계=연간 총지출/월 평균), §8-6(외화 환산·기준일), `AppSetting.currency`(기본통화)

### 맥락
- 구독은 통화별 금액(KRW/USD/EUR/JPY)을 원통화 그대로 저장.
- 미래에 **환율 API**를 붙일 예정(=모든 통화를 기본통화로 환산하는 게 종착지).
- 단, 이 앱은 **로컬/오프라인 우선**이라 네트워크에 의존하면 안 됨.
- 프로토타입 통계 화면은 전부 **₩ 단일 표기**(통화별 분리 없음). USD 서비스(Apple One·Spotify)도 ₩로 합쳐 보여줌 → 의도는 "단일 기본통화 통합".

### 결정
계산은 **통화별 집계(B 엔진)**, 화면은 **기본통화 단일 헤드라인(A 뷰)**.

```
subs → annualizedByCurrency(통화별 버킷)  →  convert(버킷, rates, base)  →  단일 ₩
        └────── B 엔진(지금 구현) ──────┘     └──── 나중에 끼울 한 스텝 ────┘
```

- B 엔진이 환산(C)의 입력 그 자체라, 환율 API가 와도 **재작업/마이그레이션 0**.
- 지금 외화가 있으면 "₩ 합계 + 외화 N건 별도 표기"로 정직하게. (틀린 하드코딩 환율로 뭉뚱그리지 않음.)

### 못박은 설계 제약
1. **엔진 = 순수 함수, 통화별 버킷 반환, 환율 모름**(rate-agnostic). 네트워크 호출 금지.
2. **환산 = `convert(buckets, rates, base)` 별도 스텝.** rates 소스가 뭐든 동일 입력.
3. **원금은 원통화 그대로 저장, 환산값 비정규화 금지.** 환산은 read-time 관심사.
4. **표시 기준통화 = `AppSetting.currency` 참조**(KRW 하드코딩 금지). 다국가 = 기본통화만 교체.

### rates 소스 3단계 (동일한 convert 스텝을 먹음)
- **(a) 없음** → 외화 별도 표기 (현재, 완전 오프라인)
- **(b) 설정에 수동 환율 입력** → 오프라인에서도 진짜 환산 (인터넷 불필요)
- **(c) 환율 API** → 같은 환율 필드 자동 채움 + 기준일

→ 오프라인 우려는 (b)를 먼저 넣어 해소 가능. (c)는 (b) 위에 얹기만 함.
→ 오프라인이어도 **기본통화 부분은 항상 계산됨**. FX는 외화를 접어 넣는 보강일 뿐, UI 형태는 동일.

### 남는 긴장
헤드라인 의미가 FX 전후로 바뀜: "₩ 구독만의 합계" → "전체의 ₩ 환산 합계". 숫자가 점프하므로 **라벨로 정직하게**("환산 포함/미포함", 기준일) 표기.

### MVP 범위
- 보여줄 것: **연간 총지출 + 월 평균**(연간/12). 단일 ₩ + 외화 별도 표기.
- cycle 정규화: WEEK×52 / MONTH×12 / YEAR×1, 모두 ÷ cycleInterval. 활성 구독 기준.
- 환율: **(a) 외화 별도 표기로 출발.** 수동 환율 입력(b)·API(c)는 **나중** (2026-06-28 결정).
- Phase 2(프로토타입의 월별 추이·유형별 구성·Top 5)는 범위 밖.
